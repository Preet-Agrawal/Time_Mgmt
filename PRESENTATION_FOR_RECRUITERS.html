Task & Time Management Web App

Problem 5 — Presentation for Recruiters

Problem Statement

Build a task management system that helps users organize tasks with deadlines and priorities.

Scope: Frontend-only (HTML, CSS, JavaScript) with persistent storage in the browser.

Our Creative Approach

  • Clean, focused UI — Single-page app: add task → see list → filter → edit/complete/delete.
  • No backend — Data lives in the browser via localStorage; no server or database.
  • Accessibility & UX — Semantic HTML, ARIA, keyboard support (e.g. Escape to close edit), responsive layout.
  • Visual clarity — Priority by color (border + badge); due dates as “Today”/“Tomorrow”; overdue highlighted.

Mandatory Features Delivered

FeatureImplementation
Add / Edit / Delete tasksForm to add; modal to edit; delete with confirmation; all sync to storage.
Priority levelsLow, Medium, High, Urgent — stored and shown with distinct styling.
Due date trackingDate picker, smart labels (Today/Tomorrow), overdue highlighting.
Filter (Today / Pending / Completed)Tabs: All, Today, Pending, Completed; instant client-side filtering.
Persistent storagelocalStorage; save on every change; data restored on load.

Tech Stack & Repository

  • HTML5 — Structure, forms, modal, list.
  • CSS3 — Variables, Flexbox, dark theme, responsive design.
  • JavaScript (ES6+) — Vanilla JS; no frameworks; clear separation of data and DOM logic.
